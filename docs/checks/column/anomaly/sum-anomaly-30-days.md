**sum anomaly 30 days** checks  

**Description**  
Column level check that ensures that the sum in a monitored column changes in a rate within a two-tailed percentile during last 30 days.

___

## **sum anomaly 30 days**  
  
**Check description**  
Verifies that the sum in a column changes in a rate within a percentile boundary during last 30 days.  
  
|Check name|Check type|Time scale|Sensor definition|Quality rule|
|----------|----------|----------|-----------|-------------|
|sum_anomaly_30_days|profiling| |[sum](../../../../reference/sensors/Column/numeric-column-sensors/#sum)|[change_percentile_moving_within_30_days](../../../../reference/rules/Percentile/#change-percentile-moving-within-30-days)|
  
**Enable check (Shell)**  
To enable this check provide connection name and check name in [check enable command](../../../../command_line_interface/check/#dqo-check-enable)
```
dqo> check enable -c=connection_name -ch=sum_anomaly_30_days
```
**Run check (Shell)**  
To run this check provide check name in [check run command](../../../../command_line_interface/check/#dqo-check-run)
```
dqo> check run -ch=sum_anomaly_30_days
```
It is also possible to run this check on a specific connection. In order to do this, add the connection name to the below
```
dqo> check run -c=connection_name -ch=sum_anomaly_30_days
```
It is additionally feasible to run this check on a specific table. In order to do this, add the table name to the below
```
dqo> check run -c=connection_name -t=table_name -ch=sum_anomaly_30_days
```
It is furthermore viable to combine run this check on a specific column. In order to do this, add the column name to the below
```
dqo> check run -c=connection_name -t=table_name -col=column_name -ch=sum_anomaly_30_days
```
**Check structure (Yaml)**
```yaml
      profiling_checks:
        anomaly:
          sum_anomaly_30_days:
            warning:
              percentile_within: 95.0
            error:
              percentile_within: 95.0
            fatal:
              percentile_within: 95.0
```
