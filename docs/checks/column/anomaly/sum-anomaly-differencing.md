**sum anomaly differencing** checks  

**Description**  
Column level check that ensures that the sum in a monitored column is within a two-tailed percentile from measurements made during the last 90 days. Use in partitioned checks.

___

## **sum anomaly differencing**  
  
**Check description**  
Verifies that the sum in a column changes in a rate within a percentile boundary during last 90 days.  
  
|Check name|Check type|Time scale|Sensor definition|Quality rule|
|----------|----------|----------|-----------|-------------|
|sum_anomaly_differencing|profiling| |[sum](../../../../reference/sensors/Column/numeric-column-sensors/#sum)|[anomaly_differencing_percentile_moving_average](../../../../reference/rules/Percentile/#anomaly-differencing-percentile-moving-average)|
  
**Enable check (Shell)**  
To enable this check provide connection name and check name in [check enable command](../../../../command-line-interface/check/#dqo-check-enable)
```
dqo> check enable -c=connection_name -ch=sum_anomaly_differencing
```
**Run check (Shell)**  
To run this check provide check name in [check run command](../../../../command-line-interface/check/#dqo-check-run)
```
dqo> check run -ch=sum_anomaly_differencing
```
It is also possible to run this check on a specific connection. In order to do this, add the connection name to the below
```
dqo> check run -c=connection_name -ch=sum_anomaly_differencing
```
It is additionally feasible to run this check on a specific table. In order to do this, add the table name to the below
```
dqo> check run -c=connection_name -t=table_name -ch=sum_anomaly_differencing
```
It is furthermore viable to combine run this check on a specific column. In order to do this, add the column name to the below
```
dqo> check run -c=connection_name -t=table_name -col=column_name -ch=sum_anomaly_differencing
```
**Check structure (Yaml)**
```yaml
      profiling_checks:
        anomaly:
          sum_anomaly_differencing:
            warning:
              anomaly_percent: 0.1
            error:
              anomaly_percent: 0.1
            fatal:
              anomaly_percent: 0.1
```
